### 用例：网关服务 72 小时长稳运行测试
一、测试概述
1. 测试目的：验证 OpenClaw 网关服务在无重启、无人工干预的情况下，连续运行 72 小时的稳定性，重点关注进程存活、端口监听、资源占用及日志健康度。
2. 测试环境：
服务端：10.0.8.10（运行网关进程）
客户端：10.0.8.15（仅按需抽样校验连接）

二、测试步骤
1. 前置准备
服务端 10.0.8.10 执行：
# 后台启动网关进程并记录 PID
nohup openclaw gateway run > gateway.log 2>&1 &
echo $! > gateway.pid
执行 exit 退出当前终端，进程在后台持续运行。
2. 定时巡检（关键节点）
在 0h（启动时）、24h、48h、72h 节点，执行巡检脚本：
./long_stability_check.sh
3. 测试收尾（72 小时结束后）
执行一键清理脚本，停止进程、备份日志并清理测试环境：
chmod +x cleanup_test.sh
./cleanup_test.sh
汇总 4 次巡检数据，分析进程存活、资源变化及日志健康度。

三、预期结果
1. 网关进程在 72 小时内全程存活，无自动退出、僵死；
2. 端口 11949 持续监听正常，客户端可正常建立连接；
3. CPU / 内存占用稳定，无持续上涨、内存泄漏现象；
4. 日志无 panic/fatal 级错误，严重错误数不随时间增长。

四、测试结论
网关服务在 72 小时内运行稳定，进程无退出、端口监听正常，资源占用平稳，日志无严重错误，长期运行稳定性达标，测试通过。
