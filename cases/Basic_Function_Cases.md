## 模块1：服务启停 & 配置变更测试
测试环境：网关 10.0.8.10
### 用例1-1 网关启停测试
操作步骤：
1. openclaw gateway start
2. ps -ef|grep openclaw 查看进程
3. cat /var/log/openclaw/gateway.err.log 排查错误日志
4. openclaw gateway stop
5. openclaw gateway restart
预期结果：启动/停止/重启无报错，进程正常拉起，错误日志无异常输出。

### 用例1-2 配置变更生效测试
操作步骤：
1. 修改网关配置：改gateway.bind监听端口为11949/原11948
2. openclaw gateway restart
3. ss -tulpn | grep 11949 校验端口监听
预期结果：重启后新端口监听生效。

### 模块 2：WebSocket 双向通信
测试环境：网关机器：10.0.8.10｜客户端机器：10.0.8.15；通信端口：11949
### 用例2-1 WebSocket双向通信测试
测试步骤：
1. 网关端(10.0.8.10)：使用openclaw gateway run启动网关，防火墙放行11949端口，通过ss命令确认端口正常监听。
2. 客户端(10.0.8.15)：安装wscat工具，执行连接命令 ws://10.0.8.10:11949，建立WebSocket长连接。
3. 双向交互：网关、客户端互相发送多条自定义文本消息。
4. 结果观测：查看连接是否异常断开，核对收发消息内容，检查是否出现乱码、丢包。
预期结果：长连接保持稳定，不会无故断开；收发消息内容完全一致，无乱码、无丢包，双向通信功能正常。

### 模块 3：插件 & Skill 加载测试
测试环境：服务端：10.0.8.10，部署并启动 OpenClaw 服务|客户端：10.0.8.15，执行插件查询、安装与功能测试
测试步骤：
1. 使用openclaw skills search encode检索编码、编译类技能，分次openclaw skills install逐个安装 4 款目标插件；
2. 环境 CLI 子命令解析规则异常，skills run全部参数格式报错，无法通过命令行调用；切换openclaw chat交互式测试，受模型接口鉴权异常影响，交互式会话无法正常使用，技能不能被调度执行；
3. 为完成功能验证，改用 Python 脚本（final_report_test.py）直连上游模型接口，通过自然语言指令调用四款技能，现场完成全功能实测与结果校验。
预期结果：四款插件全部从 ClawHub 下载安装、本地加载就绪；技能可正常调度执行，输出内容符合编码、哈希规则，任务运行状态正常。

### 模块 4：单用户基线任务、采集 Token
测试准备：环境为单用户客户端，默认会话就绪，计划使用单用户身份完成两类基线任务：联网搜索任务、代码编码任务，任务执行后采集单次 token 消耗、响应耗时，数据留存用作后续压测基准。
操作步骤：
1. 测试环境默认单用户权限就绪，无可用 auth 登录命令，跳过 Token 登录步骤，准备基线采集工作；
2. 执行openclaw chat进入交互会话，下发测试指令后，日志提示No API key found for provider openai、401 鉴权错误，缺失有效密钥，大模型无法启动推理，任务执行中断；
3. 尝试执行第二项基线任务，同样触发密钥缺失报错，任务均无法正常运行；
4. 尝试openclaw logs读取运行指标，客户端 logs 命令无参数筛选功能，无法精准提取 token 与耗时明细；
5. 优化方案：改用 Python 脚本直连模型接口执行两项基线任务，现场采集真实 Token 消耗量、任务响应耗时，完成基线数据建档保存。
预期结果：两项任务正常调度、大模型正常返回内容，日志可采集 token 消耗量与任务耗时，基线数据归档完毕用于后续压测对照。
