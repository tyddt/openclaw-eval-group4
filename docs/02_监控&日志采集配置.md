1.OpenClaw日志持久化配置
1.1 日志落地配置（两台主机：10.0.8.10 网关、10.0.8.15 客户端）

1.1.1 配置说明
因 OpenClaw 程序不支持--log-path日志指定参数、系统无预制 systemd 服务单元，采用 Shell 输出重定向方案实现日志持久化：
  >：捕获进程标准输出，存入gateway.log
  2>：捕获进程错误输出，存入gateway.err.log
末尾&实现程序后台常驻运行

1.1.2 执行部署命令
创建日志存储目录
mkdir -p /var/log/openclaw
后台启动网关+日志重定向落地
openclaw gateway run >/var/log/openclaw/gateway.log 2>/var/log/openclaw/gateway.err.log &
等待程序初始化生成日志
sleep 3
校验日志文件
ls /var/log/openclaw/

1.1.3 验收结果
执行查看命令后目录生成两个日志文件：gateway.log（正常运行日志）、gateway.err.log（异常报错日志），日志落地配置生效。

2.1.4 配置小结
本方案不依赖程序内置日志参数、不依赖 systemd 托管，兼容性强；两台业务主机使用完全相同命令完成日志配置。
