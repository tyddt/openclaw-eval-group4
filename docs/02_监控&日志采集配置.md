
1 日志落地配置（两台主机：10.0.8.10 网关、10.0.8.15 客户端）
1.1 配置说明
因 OpenClaw 程序不支持--log-path日志指定参数、系统无预制 systemd 服务单元，采用 Shell 输出重定向方案实现日志持久化：
  >：捕获进程标准输出，存入gateway.log
  2>：捕获进程错误输出，存入gateway.err.log
末尾&实现程序后台常驻运行

1.2 执行部署命令
# 创建日志存储目录
mkdir -p /var/log/openclaw
# 后台启动网关+日志重定向落地
openclaw gateway run >/var/log/openclaw/gateway.log 2>/var/log/openclaw/gateway.err.log &
# 等待程序初始化生成日志
sleep 3
# 校验日志文件
ls /var/log/openclaw/

1.3 验收结果
执行查看命令后目录生成两个日志文件：gateway.log（正常运行日志）、gateway.err.log（异常报错日志），日志落地配置生效。

1.4 配置小结
本方案不依赖程序内置日志参数、不依赖 systemd 托管，兼容性强；两台业务主机使用完全相同命令完成日志配置。

2 系统监控依赖安装与定时资源巡检
2.1 功能说明
安装系统资源监控工具，编写定时巡检脚本，每 2 小时自动采集系统时间、内存、CPU、端口信息，巡检日志统一保存至/var/log/claw_mon.log。

2.2 安装监控依赖工具
apt update && apt install -y htop dstat lsof

3.3 新建巡检脚本 ops-doc/script/claw_monitor.sh
#!/bin/bash
# 写入采集时间
echo "=====$(date)====" >> /var/log/claw_mon.log
# 采集内存使用
free -h >> /var/log/claw_mon.log
# 采集TOP前15行CPU负载
top -b -n1 | head -15 >> /var/log/claw_mon.log
# 采集全量监听端口
ss -tulpn >> /var/log/claw_mon.log

3.4 脚本部署、授权与配置定时任务
# 复制脚本至系统全局命令目录
cp ops-doc/script/claw_monitor.sh /usr/local/bin/
# 添加可执行权限
chmod +x /usr/local/bin/claw_monitor.sh
# 新增crontab定时：每2小时整点自动执行
(crontab -l;echo "0 */2 * * * /usr/local/bin/claw_monitor.sh")|crontab -

3.5 部署验证
# 手动执行脚本测试
/usr/local/bin/claw_monitor.sh
# 校验日志文件是否生成
ls /var/log/claw_mon.log
