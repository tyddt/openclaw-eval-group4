## 四、实验现象

### 1. 进程重启（restart）

- 执行 `openclaw gateway restart` 后，服务短暂中断（约 1~3 秒）
- systemd 触发服务重新拉起
- 新 Node 进程生成并重新绑定 39066 端口
- 客户端连接短暂断开后可自动恢复
- 整体恢复过程无需人工干预

---

### 2. 强制终止（kill -9）

```bash
ps -ef | grep openclaw
```
获取pid为1610431
```bash
kill -9 1610431
```
- 查看端口和进程状态
```bash
ss -lntp | grep 39066
ps -ef | grep openclaw
```

- OpenClaw 主进程被立即终止
- 39066 端口短暂释放
- systemd 自动检测并重新拉起服务
- 服务恢复后客户端自动重连成功
- 未出现持久性服务中断

---

### 3. 网络闪断（iptables）
#### 执行操作指令
```bash
# 查看网关端口监听状态
ss -lntp | grep 39066
# 添加防火墙规则，阻断外部客户端访问39066端口
sudo iptables -A INPUT -p tcp --dport 39066 -j DROP
# 校验阻断规则已生效
sudo iptables -L -n
# 删除规则，恢复网络连通
sudo iptables -D INPUT -p tcp --dport 39066 -j DROP
# 校验阻断规则已清除
sudo iptables -L -n
```

## 五、实验结论

### 1. 稳定性结论

OpenClaw 在异常场景下具备较强的自恢复能力：

- 支持进程崩溃自动恢复（kill -9 场景）

- 支持网络闪断自动重连
- 系统整体未出现资源泄漏或崩溃
