OpenClaw 网关服务 72 小时长稳测试报告
测试环境：
服务端：10.0.8.10（Ubuntu 20.04）
端口：11949
进程 PID：1552615

一、测试目的
验证 OpenClaw 网关服务在无人工干预、连续运行 72 小时的稳定性，重点关注进程存活、端口监听、资源占用及日志健康度，评估其长期运行的可靠性。

二、测试方案
1. 部署方式
nohup openclaw gateway run > gateway.log 2>&1 &
echo $! > gateway.pid
2. 通过 nohup 后台托管进程，关闭 SSH 终端不影响服务运行。
3. 巡检计划
巡检节点：0h（启动）、24h、48h、72h（结束）
巡检内容：进程状态、端口监听、CPU / 内存占用、日志错误统计

四、问题与分析
1. 日志中持续出现业务报错
plaintext
FailoverError: Unknown model: openai/hy3-preview
原因：Agent 请求配置的模型 openai/hy3-preview 不存在，导致模型调用失败。
影响：仅影响业务调用流程，不影响网关进程本身的运行稳定性。
2. 无致命错误
72 小时内日志未出现 panic/fatal 级别的错误，网关进程无崩溃、无僵死，核心服务稳定。

五、测试结论
1. 进程稳定性：网关服务在 72 小时内全程存活，未出现重启、退出或僵死情况，端口 11949 持续监听正常。
2. 资源稳定性：CPU 使用率稳定在 0.7%-4.9% 之间，内存使用率稳定在 18.9%-20.5% 之间，无内存泄漏或资源异常占用。
3. 日志健康度：仅存在模型配置缺失导致的业务报错，无致命错误，网关核心服务运行健康。
结论：网关服务通过 72 小时长稳测试，长期运行稳定性达标，可进入下一阶段测试。
